Concept

The Object-Relational Paradox

The impedance mismatch between Tally's OODBMS and SQL's Relational model is the primary hurdle. Tally uses recursive objects; SQL uses normalized tables.

Tally Object Model

  • ‚Ä¢ Hierarchical: Objects nested indefinitely.
  • ‚Ä¢ Polymorphic: Flexible schemas.

SQL Relational Model

  • ‚Ä¢ Normalized: Strict Table/Row structure.
  • ‚Ä¢ Flattened: Requires strict types.

Tally Data Model: Masters & Transactions

Tally does not expose data as SQL "tables." It exposes "Objects" or "Collections." Understanding this structure is critical. Data is broadly divided into Masters (static setup data) and Transactions (dynamic voucher entries). Explore the key objects below.

How to Find All Tables & Fields

Advanced

Tally has hundreds of internal objects not listed above. Since Tally is customizable, your specific installation might have unique "User Defined Fields" (UDFs) or Collections. Here is how to find them.

Method A: ODBC Discovery

To see exactly which tables are exposed to ODBC in your version of Tally, query the system table ODBCTables. 

import pyodbc

# 1. Connect
conn = pyodbc.connect("Driver={Tally ODBC Driver};Server=localhost;Port=9000;")
cursor = conn.cursor()

# 2. List all tables
# Query the meta-table to find available collections
cursor.execute("SELECT $Name FROM ODBCTables")
tables = cursor.fetchall()

print(f"Found {len(tables)} tables:")
for t in tables:
    print(f"- {t[0]}")

Method B: Tally.Developer 9

The official development environment (IDE) for Tally. It contains a powerful "Schema Browser".

  1. Download Tally.Developer 9 (Free from Tally Solutions).
  2. Go to Tools > Schema Browser.
  3. Search for "Voucher" or "Ledger".
  4. It lists every single Method (Field) available, including internal calculation methods and UDFs.
Why use this? It's the only way to find the exact TDL name for custom fields (e.g., _MyCustomField) added by your Tally partner.

Extraction Methods Comparison

XML over HTTP (API) Recommended

Tally acts as a web server accepting XML POST requests. Supports TDL Injection.

  • Full Data Access (Audit Log, UDFs)
  • Asynchronous & Scalable
  • Zero-footprint (No client install)

ODBC Interface

Tally acts as a database driver. Good for ad-hoc Excel queries or simple Python scripts.

  • Easy setup for simple tools
  • Very slow (Single threaded blocking)
  • No JOIN support in driver

Developer Reference: Integration Snippets

Port: 9000

Why is this the Killer Feature?

It allows you to define a temporary "Collection" optimized for SQL extraction inside the request. You don't need to install TCP files on the client's Tally machine. It supports Eager Loading (FETCH) to speed up extraction by 50x.

<ENVELOPE>
    <HEADER><TALLYREQUEST>Export</TALLYREQUEST></HEADER>
    <BODY>
        <EXPORTDATA>
            <REQUESTDESC>
                <REPORTNAME>CustomVoucherReport</REPORTNAME>
                <STATICVARIABLES>
                    <SVEXPORTFORMAT>$$SysName:XML</SVEXPORTFORMAT>
                </STATICVARIABLES>
            </REQUESTDESC>
        </EXPORTDATA>
        
        <!-- DYNAMIC TDL INJECTION START -->
        <TDL>
            <TDLMESSAGE>
                <REPORT NAME="CustomVoucherReport">
                    <FORMS>CustomVoucherForm</FORMS>
                </REPORT>
                <FORM NAME="CustomVoucherForm">
                    <PARTS>CustomVoucherPart</PARTS>
                </FORM>
                <PART NAME="CustomVoucherPart">
                    <LINES>CustomVoucherLine</LINES>
                    <REPEAT>CustomVoucherLine : MySQL_Voucher_Collection</REPEAT>
                    <SCROLLED>Vertical</SCROLLED>
                </PART>
                <LINE NAME="CustomVoucherLine">
                    <FIELDS>XML_VchNo, XML_Amount</FIELDS>
                </LINE>
                
                <!-- Temporary Collection Definition -->
                <COLLECTION NAME="MySQL_Voucher_Collection">
                    <TYPE>Voucher</TYPE>
                    <FETCH>Date, VoucherNumber, Amount, AllLedgerEntries</FETCH>
                </COLLECTION>
            </TDLMESSAGE>
        </TDL>
    </BODY>
</ENVELOPE>

Python Extraction Script

Requires pip install pyodbc. Tally must be open.

import pyodbc

# Connect to Tally ODBC (Default Port 9000)
conn_str = "Driver={Tally ODBC Driver};Server=localhost;Port=9000;"
conn = pyodbc.connect(conn_str)
cursor = conn.cursor()

# Execute Query
# WARNING: Tally ODBC is sensitive.
# 1. DO NOT use 'SELECT *' (Causes 'Could not find description' error).
# 2. DO NOT use complex fields like $PartyLedgerName if they aren't exposed.
# 3. START SIMPLE with $Date and $VoucherNumber.
query = """
    SELECT $Date, $VoucherNumber, $Narration 
    FROM Voucher 
    WHERE $VoucherTypeName = 'Sales'
"""

cursor.execute(query)
rows = cursor.fetchall()

for row in rows:
    print(f"Date: {row[0]}, Inv: {row[1]}")

conn.close()

Troubleshooting & Common Errors

Error in TDL. 'Collection:Voucher'

Cause: Using SELECT * OR selecting a field that isn't valid in ODBC.
Why: Tally tries to fetch internal/calculated methods that are context-dependent, causing a crash.
Fix: Only select specific, simple fields like $Date, $VoucherNumber. Avoid complex fields until you test them.

Supported Syntax SELECT $Name, $Parent FROM Ledger
Limitations
  • NO JOINs: You cannot JOIN tables in Tally ODBC.
  • NO GROUP BY: Aggregations must be done in Python/SQL.
  • Blocking: Long queries will freeze the Tally UI.

Target Star Schema & ETL Strategy

1. Star Schema Design

Do not replicate Tally's hierarchy in SQL. Flatten data into Facts and Dimensions.

Fact_Transaction_Lines
Granularity: One row per ledger entry. Contains Amount, LedgerKey, VoucherKey.
Dim_Ledger
Name, Group, GSTIN, State, Parent.
Dim_Date
Fiscal Year, Quarter, Month.

2. Incremental Load (CDC)

  1. Initial Load: Extract all history.
  2. Delta Logic: Query Tally for Vouchers where $AlteredOn > Last_SQL_Sync_Time.
  3. Upsert: Check if $GUID exists in SQL.
    • If exists: DELETE old lines, INSERT new lines (handles modification).
    • If new: INSERT.
  4. Concurrency: Queue exports sequentially. Tally is single-threaded.

Audit Trail & Compliance (Edit Log)

Mandated by the MCA. Tally Prime Release 2.1+ tracks the history of all changes. Extraction requires accessing deep version details using the XML API.

Forensic Fields to Extract
